## Parent

#41 — 播放页「相关推荐」：改为基于当前在播内容的同类推荐

## What to build

Close the cold-start gap. Many source titles omit `douban_id`; for those, the system now resolves the Douban subject from the title text so they still get "also-liked" recommendations, only falling back to the genre tier when even resolution fails.

## Acceptance criteria

- [ ] A pure helper (working name `parseDoubanSubjectSuggest(json)`) turns a Douban `subject_suggest` payload into a subject id (and none/empty → null), with no network dependency.
- [ ] The endpoint's resolution chain is completed: use the provided `douban_id`; if missing, resolve a subject id from the title via `subject_suggest`; then scrape also-liked (from T2). Only when no subject id can be resolved does it fall back to the genre-tag tier.
- [ ] Unit tests for the parser cover: a representative suggest payload → expected subject id, and empty/no-match → null.
- [ ] For a title with no `douban_id` but a resolvable Douban entry, the row now shows also-liked recommendations (verifiable via the resolution + parser seams).
- [ ] The graceful fallback still holds: unresolvable id + no usable genre → empty row (no generic popularity).
- [ ] `pnpm typecheck` and `pnpm test` pass.
