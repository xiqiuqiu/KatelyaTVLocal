## Parent

#41 — 播放页「相关推荐」：改为基于当前在播内容的同类推荐

## What to build

Add the primary relevance signal. For titles whose source provides a `douban_id`, the "相关推荐" row now leads with Douban's "喜欢这部的人也喜欢" (also-liked) picks for that exact title, with the genre tier demoted to a top-up. This is the enrichment that makes recommendations feel "similar to this title", not just "same genre".

## Acceptance criteria

- [ ] A pure helper (working name `parseDoubanAlsoLiked(html)`) extracts the subject page's "喜欢这部的人也喜欢" block into `DoubanItem[]`, with no network dependency.
- [ ] The endpoint, when a `douban_id` is available, fetches the Douban subject page and populates the `alsoLiked` tier from the parsed block; the genre-tag tier remains as the fallback/top-up.
- [ ] The play page passes the current title's `detail.douban_id` to the recommendation flow.
- [ ] Same caching contract as T1a (keyed by douban_id/title, CDN cache headers); a subject-page fetch failure degrades gracefully to the genre tier.
- [ ] Unit tests for the parser cover: a representative subject-page HTML snippet → expected `DoubanItem[]`, and a "block missing / markup changed" case → `[]`.
- [ ] For a title with a `douban_id`, the row visibly leads with also-liked items ordered before genre items (verifiable through the existing selection ordering contract).
- [ ] `pnpm typecheck` and `pnpm test` pass.
