
# AI找片的安全防护方案
**里程碑 1：AI 找片统一防刷底座**

目标：AI 主入口和分组入口都受同一套防刷规则控制。

- [ ] 接入 `/api/ai/find`。

- [ ] 超限时统一返回明确提示，不继续请求 AI 或资源站。

- [ ] 增加测试：用户每日次数超限应返回 429。

验收标准：

- 未登录请求 `/api/ai/find` 返回拒绝。
- 未登录请求 `/api/ai/find/group` 返回拒绝。
- 同一用户超过每日额度后，两个接口都不能继续使用。
- 测试通过：AI 找片路由测试、限额测试、类型检查。

- D1 每日额度仍然是最终精确额度来源。

**里程碑 4：AI 结果缓存和分组请求校验**

- 重复搜索同一问题不会重复调用 AI。
- 分组接口不能被随便构造请求刷资源站搜索。
- 缓存过期后行为符合预期。

**建议执行顺序**

1. 先做里程碑 1 和 2，先保护 AI 费用。
2. 再做里程碑 3，安全开放注册。
3. 再做里程碑 4，降低重复成本。
4. 最后做里程碑 5 和 6，完善后台和上线验证。

第一版上线的最低可接受范围是：**里程碑 1 + 2 + 3**。这三项完成前，不建议同时开放注册和 AI 找片。
